Lesson 19: Java Memory Management

1. Stack vs Heap

- Stack: Stores primitive variables and references. Fast, automatically managed. - Heap: Stores objects. Managed by Garbage Collector.

2. Object Lifecycle

  • Creation with new
  • Reachability & references
  • Eligible for garbage collection when no references exist

3. Garbage Collection (GC)

Java automatically cleans up unused objects using GC.


MyClass obj = new MyClass();
obj = null; // eligible for GC
System.gc(); // request GC

4. Common Issues

  • Memory leaks via lingering references
  • Excessive object creation in loops
  • Large collections not cleared

Task

Write a program creating multiple objects and nullify references to see GC in action (use System.gc()).
